# Admin Guide

This section covers platform administration tasks you can perform through
the I4G Console. You need the **admin** role.

## What admins can do

| Task                                                        | What it covers                               |
| ----------------------------------------------------------- | -------------------------------------------- |
| [User Management](user-management.md)                       | Add users, change roles, deactivate accounts |
| [Scheduled Reports](scheduled-reports.md)                   | Automate recurring report generation         |
| [Partner Feed](partner-feed.md)                             | Configure indicator feeds for partners       |
| [Site Investigation Settings](site-investigation-config.md) | Configure SSI scan behavior                  |
| [Auto-Investigation](auto-investigation.md)                 | Set up automatic URL investigation           |

## Roles overview

I4G uses role-based access control with four primary roles:

| Role        | What they can do                                                |
| ----------- | --------------------------------------------------------------- |
| **user**    | Submit intake reports, view own submissions.                    |
| **analyst** | Review cases, access Console tools, participate in engagements. |
| **manager** | Everything analysts can do, plus create and manage engagements. |
| **admin**   | Full access — user management, archival, system configuration.  |

For detailed role and permission information, see
[Access & Roles](../security/access-and-roles.md).
